## 1. 现在还常用吗？

**在典型业务开发里：不多。**  
写订单、CRUD、Spring 服务时，更常见的是 `boolean`、枚举、`Set`、数据库字段，很少直接写 `& | <<`。

**在偏底层 / 性能敏感的地方：仍然很多。**

| 领域 | 例子 |
|---|---|
| 系统与运行时 | 页表、fd_set、GC 标记、对象头标志位 |
| 网络与协议 | TCP flags、HTTP/2 frame flags、压缩编码 |
| 存储与数据库 | 位图索引、NULL 位图、权限位 |
| 图形 / 游戏 | 颜色通道打包、碰撞层 mask |
| 算法与面试 | 状态压缩 DP、位集合枚举 |
| 本仓库 | Disruptor `availableBuffer`、权限 `READ\|WRITE` |

可以记成：

```text
业务层：少手写，但不等于「过时」
基础设施：天天在用，只是被库包起来了
```

你用 `EnumSet`、Redis `SETBIT`、文件权限 `rwx`，底层常常就是位运算。

## 2. 基本原理：数在内存里是一排开关

计算机存整数，最小单位是 **bit（位）**：只能是 `0` 或 `1`。

一个 Java `int` 有 32 位，`long` 有 64 位。可以把它们想成一排开关：

```text
十进制 13

二进制（低 8 位示意）：  0000 1101
下标（从右、从 0）：     7654 3210
含义：                   8+4+0+1 = 13
```

**位运算不做「加减乘除那种算术故事」**，而是：**按位、彼此独立地改这些 0/1**。  
每一位和其他位互不影响（除非你故意用移位把信息挪位置）。

这就是全部理论基础：

> 数 = 一排开关；位运算 = 规定好的拨开关规则。

## 3. 六种核心运算

下面用 4 位示意（真实是 32/64 位，规则相同）。

### 3.1 `&` 按位与：都是 1 才是 1

```text
  1101
& 1011
= 1001
```

人话：**两排开关都开着的位置，结果才开。**  
用途：遮罩查询、保留某些位、判断某位是否为 1。

### 3.2 `|` 按位或：有一个 1 就是 1

```text
  1100
| 0010
= 1110
```

人话：**有一边开着就开。**  
用途：打开某些位（置位）、合并标志。

### 3.3 `^` 按位异或：相同为 0，不同为 1

```text
  1100
^ 1010
= 0110
```

人话：**两排不一样的位置结果为 1。**  
用途：切换某一位（开↔关）、简单校验、部分交换技巧。

### 3.4 `~` 按位取反：0↔1

```text
~ 0010  →  ...1101   （其它高位也会翻，注意符号位）
```

人话：**全部开关扳反。**  
用途：常和 `&` 组合做「清某一位」：`x & ~mask`。

### 3.5 `<<` 左移：整体往左挪，右边补 0

```text
  0001 << 3  =  1000     // 1 变成 8
```

人话：**开关阵列往左推，空位补 0。**  
对正数：`x << n` 约等于 `x * 2^n`。  
用途：造「只有某一位是 1」的 mask：`1 << k`。

### 3.6 `>>` / `>>>` 右移：整体往右挪

```text
  1000 >> 3   =  0001     // 算术右移（有符号，左侧补符号位）
  1000 >>> 3  =  0001     // 逻辑右移（无符号，左侧补 0）
```

对正数：`x >>> n` 约等于 `x / 2^n`（整数除）。  
位图里算「第几个 long」用 `id >>> 6`（÷64）。

## 4. 组合套路：置位、清位、切换、判断

这四套是实际代码里 90% 会用到的：

```java
int flags = 0;
int BIT2 = 1 << 2;   // 第 2 位的面具：...0100

// 置位：打开第 2 位
flags |= BIT2;       // flags = flags | BIT2

// 判断：第 2 位开了吗？
boolean on = (flags & BIT2) != 0;

// 清位：关掉第 2 位
flags &= ~BIT2;      // flags = flags & ~BIT2

// 切换：开变关、关变开
flags ^= BIT2;
```

逐步看「清位」为什么是 `& ~`：

```text
BIT2 = 0100
~BIT2 ≈ ...1011     // 只有第 2 位是 0
flags & ~BIT2       // 强制第 2 位变 0，其它位原样
```

## 5. 和普通算术的关系

| 位运算 | 常等价于（正数、不溢出时） |
|---|---|
| `x << n` | `x * (2^n)` |
| `x >>> n` | `x / (2^n)` |
| `x & (2^n - 1)` | `x % (2^n)`，如 `x & 63` ≡ `x % 64` |
| `x & -x` | 取出最低的那个 1（进阶技巧） |

所以位图里的：

```text
word = id >>> 6      // id / 64
bit  = id & 63       // id % 64
```

不是玄学，就是**用移位/掩码做除法和取余**，因为除数恰好是 2 的幂，CPU 上极快。

## 6. 小结

```text
原理：整数 = 一排 0/1 开关；位运算 = 按位拨开关
常用四件套：| 打开，& 检查，&~ 关掉，<< 造面具
业务代码：少手写；库和系统里：仍然到处都是
学它：为了读懂底层、写位图/标志位/协议，不是为了天天炫技
```
